# CLAUDE.md

## Stack

- **Backend:** Go 1.25, Chi v5 router, Jet ORM, oapi-codegen, gorilla/sessions, goose migrations
- **Frontend:** React 19, Vite (Bun), TanStack React Query, shadcn/ui, Orval for OpenAPI codegen
- **Database:** PostgreSQL with go-jet for type-safe queries
- **Module name:** `saas-template`

## Commands

```bash
# Backend (from backend/)
bash commands.sh webserver          # Start dev server
bash commands.sh openapi:codegen    # Regenerate all API types + frontend Orval
bash commands.sh migration:codegen {name}  # Create new migration
bash commands.sh migration:up       # Run migrations + regenerate Jet models
bash commands.sh lint               # Lint
bash commands.sh test               # Test

# Frontend (from frontend/)
bun run dev                         # Start dev server
bun run build                       # Type check + build
bunx orval                          # Regenerate API hooks from OpenAPI specs
```

## Three API Tiers

| Tier | Spec file | Base URL | Package | Auth |
|------|-----------|----------|---------|------|
| Public | `openapi-public.yaml` | `/api/public/v1` | `oapi_public` | None (rate limited) |
| Authenticated | `openapi.yaml` | `/api/v1` | `oapi` | Session required |
| Admin | `openapi-admin.yaml` | `/api/admin/v1` | `oapi_admin` | Session + admin role |

## Adding a New Endpoint

1. Add path + `operationId` to the correct OpenAPI spec
2. Run `bash commands.sh openapi:codegen`
3. Create handler file — thin, delegates to app_service
4. Create app_service file — business logic, one file per operation
5. Add repository (read) or mutation (write) functions if needed
6. Add `audit.Log()` for security-relevant actions

## Naming Rules

The `operationId` drives everything:

```
operationId: post-api-v1-signin
  → Method:  PostApiV1Signin
  → File:    post_api_v1_signin.go
```

- Method name: split by hyphens, PascalCase each segment
- File name: replace hyphens with underscores, lowercase

## Layer Responsibilities

- **Handler** — Thin. Extract request body, call app_service, map errors to typed response objects. Session creation/destruction happens here, not in app_service.
- **App Service** — Business logic. Gets session via `middleware.GetSessionData(ctx, app.Session())`. Calls repository/mutation. Calls `audit.Log()`. One file per operation.
- **Repository** — Read-only queries (Jet ORM). Functions prefixed with `Get`.
- **Mutation** — Write operations (Jet ORM). Functions prefixed with `Create`/`Update`/`Delete`. Always set `CreatedAt`/`UpdatedAt`.

## Error Handling

- Expected errors (401, 400): return typed response object (`PostApiV1Signin401JSONResponse{...}`)
- Unexpected errors: return `nil, err` — triggers 500 via middleware (logs internally, returns generic message)
- Never leak internal error details to the client

## Session Flow

Context injector stores `*http.Request` in context → app_service calls `middleware.GetSessionData(ctx, app.Session())` → returns `SessionData{UserID, TenantID, Role}` as strings → parse to `uuid.UUID` before use.

## Generated Code

- `generated/oapi/` — Do not edit. Regenerated by `openapi:codegen`.
- `generated/db/` — Do not edit. Regenerated by `migration:up`.
- `frontend/src/services/api/v1*.ts` — Do not edit. Regenerated by Orval.
